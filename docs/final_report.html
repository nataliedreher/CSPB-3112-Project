Final Report



Overview Of Project Progression

At the start of this class I struggled a bit with what kind of project I wanted to pursue. I settled on learning a new language and framework. To that end I decided to make an an Android app using Java as the backend language. I decided on a text based game because I wanted a fun project. I started this journey by auditing the following course from Couresa: java-introduction. Some of my progress came to a halt when my work schedule became on manageable. After solving that problem I finished up the Java course and began work on Android Studio. To get started on Android Studio I began with the resources from Android.com. After running through the full tutorial I began work on my game. Initially I was going to just make a standard adventure game but after a while I decided on something different. In the game you play as a bartender at a tavern in a fantasy setting. You start by entering in some demographics. After that the game starts with your mother informing you that you she owes 500 gold and if you don't come up with the money in 5 days you will lose your home. Your job is to try to earn good enough tips via bantering with your patrons make the needed gold. As far as the mechanics of the implementation goes I got almost everything I decided I wanted to included. What I planned on including that made it into the current release:

  1. The ability to enter and save player information through a single play through.
  2. A branching dialogue structure leading to differing tip amounts.
  3. Randomized patron selection from a predefined cast.
  4. A method to track the day and the amount of gold a player has.
  5. A method to track if the player has made enough gold.

What didn't quite make the cut:

  1. Full use of demographics. Currently only the name is used in game.
  2. While the dialogue tree structure was finished I ran out of time to generate the actual dialogue as using ChatGPT ended up being very unwieldy to generate 18-27 lines of dialogue per character for 21 separate characters.

What did I actually learn?

  • I learned a lot about how Android Studio works.
    • For the most part Android Studio in very intuitively built and there is a robust community of developers so it's very easy to find solutions to quirks with the IDE.
  • I learned a lot about Android code structure
    • Android is very tight in how different files/classes share methods and variables.
  • I learned that Android development is very satisfying.
    • It's really cool to implement something then immediately be able to test it out.
  • I learned that it's incredibly easy to get null pointer errors in Java.

Project Assessments

  • Complete... sort of. The largest measure of success for my project will be to have a completed app that is available in the android app store.
    • While the game is not published on the play store it is available on my github for download. I want to have the game slightly more finished before publishing on an official platform.
  • Complete. Another important measure of success will be ensuring the app works fairly well and is at least somewhat presentable.
    • What functionality that is implemented works very well and while the app isn't gorgeous by any means but it's not too shabby.
  • Complete?The last measure of success is for the app to be at least somewhat fun to play. No one wants to play a terrible game.
    • This is very subjective but I think the concept is fun and when it's fully implemented it will be a fun way to play around for 10-20 minutes.

The biggest takeaway I have from this project is that using building an Android app to learn Java isn't the best was to go about things. While I do think I gained a very noticeable amount of skill in Java I learned so much more about Android app development that this project became mostly about learning Android development. That being said I don't regret the choice to make an Android app however if I could do it again I would've had the goal to learn about app development with learning Java being more of a fringe benefit.
